Issues We Encountered

1) Assigning roles - We had 4 people so we had to figure out a way we could all share our changes. We used Google Drives to solve this issue and had all our code in a Google Doc.

2) The "submit" algorithm we had was not working as planned. We could not find a way to output our user's input to the Sign page. [SOLUTION] We found out the reason for this was because we were not a hosted website on a server. Therefore the PHP code we had created was deemed useless. Our solution was to use pre-made template code to give an illusion on how our Sign web page was supposed to look.

3) Right now, there is no actual way of preventing one person from signing a petition more than once. There will be a solution in the future, but for now we have made a seperate page that redirects the user to a thank you page after signing a petition.

4) The Counter for a single petition has been removed from the public. This was done to prevent the user from non stop clicking the "Sign this Petition!" button and instead making the process longer. We can't seem to find a way to make it stay on a webpage, but it is stored in the websites code and do not reset until the website is demolished.
